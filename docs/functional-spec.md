# 기능 명세

## 문서 목적
- 현재 대시보드 구현을 기준으로 웹서비스 MVP 기능을 명세한다.
- 현재 명세는 프로토타입 보존 중심이다.

## 서비스 한 줄 정의
- 사용자가 미국 주식 티커를 입력하면, 최근 시세와 기술적 지표를 바탕으로 참고용 매매 시그널을 보여주는 대시보드 웹서비스.

## 핵심 사용자
- 단기 스윙 관점으로 종목 흐름을 빠르게 보고 싶은 사용자
- 복잡한 HTS/MTS 대신 간단한 시각 요약을 원하는 사용자

## 핵심 사용자 시나리오
1. 사용자가 대시보드에 접속한다.
2. 기본 티커가 자동 조회된다.
3. 현재가, 등락률, 시그널, 차트를 본다.
4. 다른 티커를 입력하고 조회한다.
5. 실패 시 fallback 데이터를 기반으로 화면은 유지된다.
6. 60초 후 데이터가 자동 갱신된다.

## 화면 명세
### 1. 헤더 영역
기능:
- 서비스 성격 설명
- 현재 티커와 현재가 표시
- 마지막 업데이트 시각 표시
- 티커 입력
- 조회 버튼
- 조회 상태 메시지

입력 규칙:
- 영문 대문자 기준 티커 입력
- 공백 제거 후 조회
- Enter 키로도 조회 가능

예외:
- 빈 입력은 조회하지 않음
- 조회 중 중복 요청 방지

### 2. 티커 요약 카드
기능:
- 티커 심볼 표시
- 현재가 표시
- 전 캔들 대비 등락액 표시
- 전 캔들 대비 등락률 표시
- 상승/하락에 따라 색상 변경

계산:
- `delta = last.close - prev.close`
- `pct = delta / prev.close * 100`

### 3. 실시간 매매 시그널 카드
기능:
- 시그널 문구 표시
- 점수 표시
- 5단계 시그널 미터 표시
- RSI 해석 문구
- MA 추세 해석 문구
- 볼린저 위치 해석 문구

시그널 단계:
- 1: 강력 매도
- 2: 매도
- 3: 보유
- 4: 매수
- 5: 강력 매수

점수 구간:
- `score >= 5`: 강력 매수
- `score >= 2`: 매수
- `-2 < score < 2`: 보유
- `score > -5`: 매도
- 그 외: 강력 매도

### 4. 가격 차트 영역
기능:
- 캔들 차트 표시
- MA20 라인 표시
- MA50 라인 표시
- 볼린저 상단/하단 표시
- 거래량 히스토그램 표시
- 최초 조회 또는 티커 변경 시 fit-content 수행

데이터 최소 조건:
- 60개 이상 캔들 필요

### 5. 지표 카드 영역
기능:
- RSI(14)
- MACD(12,26,9)
- Signal/Histo
- Volume(20 평균)

표시 규칙:
- 값이 없으면 `-`
- 소수점 형식 유지

### 6. 유의문구 배너
기능:
- 투자 참고용 문구 고정 노출

규칙:
- 수익 보장처럼 해석될 수 있는 문구 금지

## 데이터 소스 명세
### 1차 데이터 소스
- Yahoo Finance chart endpoint
- interval: `1h`
- range: `3mo`

### 실패 대응
- 직접 호출 실패 시 대체 경로 시도
- 모두 실패 시 mock 데이터 생성

fallback 정책:
- 서비스는 최대한 화면을 유지한다.
- fallback 상태임을 사용자에게 명시한다.
- 운영 환경에서는 fallback 사용 여부를 로그로 남긴다.

## 계산 명세
### SMA
- 기간 평균
- MA20, MA50 계산에 사용

### EMA
- MACD 계산 기반

### RSI(14)
- 기본 기간 14
- 평균 상승폭/하락폭 기반

### Bollinger Bands(20, 2)
- basis: SMA20
- upper/lower: basis ± 2 * 표준편차

### MACD(12, 26, 9)
- `ema12 - ema26`
- signal: `macd`의 EMA9
- histo: `macd - signal`

### 평균 거래량
- 최근 20개 데이터 평균

## 시그널 계산 명세
### RSI 반영
- `<= 25`: +3
- `<= 35`: +2
- `< 45`: +1
- `< 55`: 0
- `< 65`: -1
- `< 75`: -2
- 그 외: -3

### MA 및 추세 반영
- `price >= ma20`: +1, 아니면 -1
- `ma20 >= ma50`: +1.5, 아니면 -1.5
- `ma20 >= ma20Prev`: +0.5, 아니면 -0.5
- `macd >= macdSignal`: +1, 아니면 -1
- `macdHist >= 0`: +0.7, 아니면 -0.7

### 볼린저 반영
- `price < bbLower`: +2
- `price > bbUpper`: -2
- 하단 25% 이내: +0.8
- 상단 75% 이상: -0.8
- 중간 영역: 0

### 단기 가격 반영
- `price >= prevPrice`: +0.5
- 아니면 -0.5

### 거래량 반영
- `volume / avgVolume >= 1.4` 이고 상승이면 `+0.8`
- `volume / avgVolume >= 1.4` 이고 하락이면 `-0.8`
- `<= 0.6` 이면 가산점 없음

### 점수 마감 규칙
- 최종 점수는 `-10 ~ 10` 범위로 clamp
- 소수 첫째 자리까지 반영

## 비기능 명세
### 성능
- 초기 렌더 후 조회 응답은 체감상 빠르게 보여야 한다.
- 차트 렌더는 일반 노트북/모바일 브라우저에서 무리 없어야 한다.

### 안정성
- 외부 API 실패만으로 전체 화면이 깨지면 안 된다.
- 데이터 부족 시 명시적 오류 메시지를 보여야 한다.

### 접근성
- 입력창과 버튼은 키보드 접근 가능해야 한다.
- 색상만으로 상태를 구분하지 않도록 텍스트를 유지한다.

### 반응형
- 모바일에서도 주요 카드와 차트가 세로 흐름으로 보여야 한다.

### 정책
- 투자 참고용 정보임을 명확히 노출한다.
- 투자 자문으로 해석될 수 있는 표현을 피한다.

## MVP 완료 기준
- 기본 티커 진입 조회 가능
- 티커 변경 조회 가능
- 정상 조회 시 차트/카드/시그널 표시
- 실패 시 fallback 동작
- 자동 새로고침 동작
- 콘솔 치명 에러 없음
- 유의문구 노출 유지
